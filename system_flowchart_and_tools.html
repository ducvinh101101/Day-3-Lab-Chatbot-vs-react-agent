User Message (Tin nhắn người dùng)

Giao diện Web gửi tin nhắn qua phương thức POST tới /api/chat. Người dùng có thể gửi câu hỏi ngắn chào hỏi, câu hỏi ngoài lề hoặc yêu cầu tìm phòng, giữ chỗ, xác nhận.

Guardrail Filter (Bộ Lọc Phạm Vi & Phân Loại Cố Định)

Hệ thống sẽ thực hiện khớp chuỗi nhanh (Regex và so sánh bộ từ khóa) trước khi gọi LLM:

  • Chào hỏi thuần túy: Nếu chỉ gửi "hello", "chào bạn", bot sẽ chào lại kèm theo mô tả phạm vi của mình ngay lập tức.
  • Truy vấn mã phòng/giá cụ thể: Nếu hỏi giá phòng đã có trong danh sách tìm kiếm trước đó, hệ thống sẽ trích xuất số tiền/mã phòng và mô tả trực tiếp mà không cần chạy mô hình ngôn ngữ.
  • Ngoài phạm vi: Nếu hỏi thời tiết, lịch sử, toán học... hệ thống sẽ trả về câu từ chối mẫu để bảo đảm tính chính xác 100%.

Agent Router (Trình Điều Hướng Tác Vụ)

Tùy thuộc vào sự hiện diện của tệp mô hình GGUF local Phi-3-mini-4k-instruct-q4.gguf tại thư mục models/:

  • Nếu có mô hình, bot sẽ khởi tạo LocalProvider và chạy vòng lặp suy luận ReAct.
  • Nếu không có mô hình, bot tự động hạ cấp xuống luồng xử lý bằng luật (RuleBasedVinpearlAgent) để kiểm thử và phát triển nhanh.

Rule Fallback (Bản Rút Gọn Bằng Luật)

Nếu ở chế độ Rule-based, hệ thống tự động bóc tách các trường thông tin thông qua biểu thức chính quy (Regex):

  • Trích xuất ngày tháng (định dạng YYYY-MM-DD hoặc DD/MM/YYYY)
  • Trích xuất số lượng người lớn, trẻ em.
  • Trích xuất giá tối đa (dưới 6 triệu, không quá 10 triệu...).
  • Tự động gọi các API local trong Toolset tương tự như Agent nhưng theo kịch bản cứng vững.

ReAct LLM Agent (Tác Tử Suy Luận Thông Minh)

Sử dụng mô hình ngôn ngữ kết hợp với định dạng phản hồi quy chuẩn:

  • Thought: LLM viết ra suy nghĩ để chuẩn bị bước tiếp theo.
  • Action: LLM phát ra lời gọi hàm (ví dụ: search_vinpearl_packages(location="Nha Trang", ...)).
  • Observation: Backend bắt được cú pháp Action, chạy code Python thật và đưa kết quả trả lại vào prompt của LLM.
  • Quá trình lặp lại tối đa 6 bước trước khi kết luận bằng Final Answer:.

Vinpearl Toolset (Tệp tin Công Cụ Đăng Ký)

Đây là tập hợp các API thật kết nối trực tiếp với Cơ sở dữ liệu Inventory dạng JSON. Tác tử LLM hoặc Công cụ so khớp luật đều giao tiếp qua các phương thức Python cụ thể tại đây để cập nhật số lượng phòng và lấy kết quả trả về cho khách.
